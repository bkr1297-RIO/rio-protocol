# RIO Protocol Simulator Artifacts

Reference artifacts generated by the **RIO Protocol Simulator v1.0 (Ed25519)**. Every hash and signature in these files is cryptographically valid and recomputable from the artifact fields — nothing is mocked.

These artifacts serve as **test vectors** for SDK development, conformance testing, and protocol education.

---

## Generation Modes

The simulator (`simulate.py` at repo root) supports four generation modes:

| Mode | Command | Output |
|------|---------|--------|
| Valid artifacts | `python simulate.py --generate-valid` | 7 artifacts + 4/4 verification PASS |
| Corrupt signature | `python simulate.py --generate-invalid-signature` | Intent with wrong-key signature (Stage 2 rejection) |
| Tampered ledger | `python simulate.py --generate-tampered-ledger` | 3-entry chain with row 2 tampered — verification FAIL |
| Full example | `python simulate.py --generate-full-example` | Narrated 8-stage protocol flow with all artifacts |
| All modes | `python simulate.py --all` | Runs all four modes |
| Verify | `python simulate.py --verify <file.json>` | Verify any artifact |

## Cryptographic Suite

| Component | Algorithm | Detail |
|-----------|-----------|--------|
| Intent signing | Ed25519 | 64-byte signature, base64-encoded |
| Execution token | Ed25519 | Signs `intent_id|nonce` |
| Receipt hash | SHA-256 | Over canonical JSON of 8 receipt fields |
| Entry hash | SHA-256 | Over pipe-delimited seal_data string |
| Ledger hash | SHA-256 | `SHA256(prev_hash|entry_hash|timestamp)` |
| HMAC signature | HMAC-SHA256 | `HMAC(service_token, entry_hash|ledger_hash|timestamp)` |
| Public key fingerprint | SHA-256 | First 16 hex chars of `SHA256(raw_32_byte_public_key)` |

---

## Verification Checks (4 total)

1. **receipt_hash_integrity** — Recompute receipt_hash from source fields
2. **execution_ledger_chain_integrity** — Recompute entry_hash per row, verify prev_hash linkage from GENESIS
3. **post_exec_ledger_hash_integrity** — Recompute ledger_hash per row
4. **post_exec_hmac_signatures** — Recompute HMAC-SHA256 per row using service token

## Hash Formulas

```
intent_id         = SHA256("{intent}|{source}|{timestamp}")
parameters_hash   = SHA256(canonical_json({intent, source, model, timestamp, intent_id}))
receipt_hash      = SHA256(canonical_json({intent_id, model, agent, source, response_chars,
                                           signature_hash, result_hash, timestamp}))
entry_hash        = SHA256("{action}|{agent}|{approver}|{executed_by}|{intent_id}|
                            {parameters_hash}|{result}|{reason}|{receipt_hash}|
                            {prev_hash}|{timestamp}")
ledger_hash       = SHA256("{prev_hash}|{entry_hash}|{timestamp}")
hmac_signature    = HMAC-SHA256(service_token, "{entry_hash}|{ledger_hash}|{timestamp}")
canonical_json    = json.dumps(obj, separators=(',',':'), sort_keys=True).encode('utf-8')
```
