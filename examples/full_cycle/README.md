## Execution Flow

### Step 1 — Intent Signing (3.15 ms)
The intent "Transfer 500 units from treasury to operations account" is signed using **ECDSA secp256k1** with the gateway's private key. The signature is DER-encoded and base64-wrapped.

### Step 2 — Gateway Verification and Routing (6.26 ms)
The Sovereign Gate verifies the signature against the authorized public key. Invalid or unsigned intents are **rejected** (fail-closed). The intent is then routed to the specified model.

### Step 3 — Model Execution (6,005.87 ms)
The Claude model processes the intent. The response is captured and its hash is computed for the receipt.

### Step 4 — Ledger Commit (0.11 ms)
A v2 receipt is generated containing `intent_hash`, `action_hash`, `verification_hash`, and `verification_status`. The receipt is committed to the append-only ledger with a `prev_hash` linking to the previous entry.

### Verification — Independent Check (PASS)
All six verification checks pass:
1. Receipt hash is valid SHA-256
2. Intent hash is valid SHA-256
3. Ledger chain integrity verified (GENESIS → entry 1 → entry 2)
4. ECDSA secp256k1 signature verified
5. Timestamp ordering correct
6. Fail-closed enforcement confirmed via audit log
